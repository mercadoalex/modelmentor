# Contributing to ModelMentor

Thank you for considering contributing to ModelMentor! Please follow these guidelines to help us maintain a high-quality, stable, and secure project.

## Getting Started

1. **Fork the repository** and clone it to your local machine.
2. **Install dependencies** using `npm install`.
3. **Create a new branch** for your feature or bugfix.

## Coding Standards

- Use clear, descriptive commit messages.
- Write clean, readable, and well-commented code.
- Follow the existing code style and formatting.
- Add or update tests as appropriate.

## Dependency Policy

- **Do NOT add unnecessary, low-quality, or poorly maintained npm packages (“stupid packages”).**
- Only add dependencies that are well-maintained, widely used, and essential for project functionality.
- Always check for recent updates, community adoption, and security before adding a new package.
- Avoid adding packages for trivial tasks that can be accomplished with native JavaScript or existing dependencies.
- All new dependencies must be justified in your pull request and are subject to review.

## Pull Requests

- Ensure your branch is up to date with `main` before submitting a pull request.
- Describe your changes clearly in the PR description.
- Reference related issues if applicable.
- Make sure all tests pass before requesting a review.

## Reporting Issues

- Use the issue tracker to report bugs or request features.
- Provide as much detail as possible, including steps to reproduce, expected behavior, and screenshots if relevant.

## Code of Conduct

- Be respectful and inclusive.
- Constructive feedback is welcome; personal attacks are not tolerated.

Thank you for helping make ModelMentor better!
